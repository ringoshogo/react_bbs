■Django-restframeworkの作成方法

参考URL：https://tutorial.djangogirls.org/ja/django_installation/

●仮想環境の構築
・空のフォルダでvsCodeを開く
・仮想環境の構築
⇒ C:\Users\Name\djangogirls> python -m venv myvenv
・仮想環境の起動
⇒ C:\Users\Name\djangogirls > myvenv\Scripts\activate
●Djangoのインストール
・requirements.txt ファイルを新規作成
(requirements.txtにはpip install でインストールする依存関係の一覧)
djangogirls
├── myvenv
│   └── ...
└───requirements.txt
・
・requirements.txt ファイルに下記を追記
Django~=2.2.4
・Djangoのインストール
pip install -r requirements.txt
●プロジェクトの新規作成
(myvenv) C:\Users\Name\djangogirls> django-admin.exe startproject mysite .
※ settings.pyファイルには、ウェブサイトの設定が含まれています。
●設定の変更
settings.py/
※ 言語の設定
TIME_ZONE = 'Asia/Tokyo'
LANGUAGE_CODE = 'ja'

※静的ファイルのパスを追加
STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'static')

※ DEBUG が True に設定されていて、ALLOWED_HOSTS が空のリストの時は、
自動的に ['localhost', '127.0.0.1', '[::1]'] という3つのホストに対してチェックが行われます。
⇒ デプロイして使用するホストネームを追加する場合、以下の追記
ALLOWED_HOSTS = ['127.0.0.1', '.pythonanywhere.com']

●DBのセットアップ
(myvenv) ~/djangogirls$ python manage.py migrate

●アプリケーション作成
(myvenv) C:\Users\Name\djangogirls> python manage.py startapp blog
・Djangoにアプリケーションを伝える
settings.py
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'blog.apps.BlogConfig', // 新規追加 -> アプリケーション名.apps.[apps内のクラス名]
]

セットアップ終了
